The AI Bookkeeper Setup: Reconciled Books Every Day - Automatically

By Gideon Spencer · March 27, 2026

This guide walks you through setting up an AI that opens your accounting software every single day, goes through your transactions, categorizes everything it knows, and sends you a daily report - including a list of any transactions it has questions about.

You do not need to be technical to follow this. Each step tells you exactly what to click and what to type. By the end, your books will be reconciling themselves daily while you focus on running your business.

What This Builds

An AI agent that runs every day on a schedule. It opens Chrome, navigates to your accounting platform (QuickBooks, Xero, Wave, or others), reviews new transactions, applies your categorization rules, and generates a daily report showing exactly what was done and what still needs your attention. The reconciliation happens directly inside your accounting platform - not in a separate log.

Before You Start: What You Need

Claude Desktop App

The free app from Anthropic. You'll also need a Claude Pro subscription ($20/month) to use Claude Code and the Chrome extension.

Google Chrome

Chrome must be your browser. The AI uses a Chrome extension to actually operate inside your accounting platform.

Accounting Platform Login

You must be logged into QuickBooks Online, Xero, Wave, or FreshBooks in Chrome before the daily automation runs. If you're logged out, the AI will stop and alert you.

A Mac Computer

The daily scheduling uses a Mac-specific tool (launchd). Windows instructions are at the bottom of this guide.

Important - Before Every Run

Stay Logged Into Your Accounting Platform

The AI cannot log in for you. Before each daily session, your accounting platform must already be open and logged in inside Chrome. If it's not, the automation will pause and send you an alert telling you to log in and try again. Set your accounting platform to "keep me logged in" or "remember me" to avoid this.

Step-by-Step Setup

Step 1

Download the Claude App and Subscribe to Claude Pro

Go to claude.ai/download on your Mac. Download the Claude desktop app and install it like any other Mac application (drag it to your Applications folder when prompted).

Open the app and sign in or create a free account. Then go to claude.ai/settings in your browser and subscribe to Claude Pro ($20/month). You need Pro to use Claude Code and the Chrome extension.

Once subscribed, the Claude desktop app will unlock the tools used in this guide.

Step 2

Open Claude Code

Claude Code is the part of Claude that can run tasks on your computer automatically - including setting things up on a daily schedule. Think of it as giving Claude the ability to actually do things, not just talk about them.

To open it:

  1. Press Command + Space on your keyboard to open Spotlight search
  2. Type Terminal and press Enter - a black window will open
  3. In that black window, type claude and press Enter
  4. Claude Code will launch. You'll see a prompt that says you can start typing

This black window is where you'll paste the setup commands below. You only need to do this setup once.

Step 3

Install the Claude Chrome Extension

Step 5

Run First-Time Setup

Make sure you're logged into your accounting platform in Chrome, then type this in Claude Code and press Enter:

/bookkeeper

The first time you run it, Claude will ask you five quick questions to configure your setup - things like what platform you use, what vendors should be auto-categorized, and where to save your daily reports. Takes about two minutes.

After that, every daily run skips this setup and goes straight to reconciling your books.

# AI Bookkeeper Skill

You are an automated daily bookkeeper. When invoked, you use the browser
to open the user's accounting platform, review new transactions, apply
categorization rules, reconcile directly inside the platform, and produce
a daily report for the business owner.

---

## REQUIREMENTS CHECK

Before doing anything else:

1. Check whether ~/.bookkeeper-config.json exists.
   - If it does NOT exist, run FIRST-TIME SETUP below.
   - If it exists, load the config and proceed to DAILY RECONCILIATION.

2. Check that Chrome is open and the user is logged into their accounting
   platform. Navigate to the platform URL from config. If a login screen
   appears, STOP and output this message exactly:

   "ACTION REQUIRED: [Platform name] is showing a login screen in Chrome.
   Please log in manually, then type /bookkeeper to run again."

   Do not attempt to log in for the user. Do not proceed past a login wall.

---

## FIRST-TIME SETUP

Ask the following questions one at a time and save all answers to
~/.bookkeeper-config.json when complete.

Question 1:
"What accounting platform do you use?
  (a) QuickBooks Online
  (b) Xero
  (c) Wave
  (d) FreshBooks
  (e) Other - type the name"

Question 2:
"What is the URL you use to log into it?
For example: app.qbo.intuit.com for QuickBooks Online."

Question 3:
"What currency does your business primarily operate in?
For example: USD, CAD, GBP."

Question 4:
"List any vendors or transaction sources you want auto-categorized
without being asked. Use this format:
  Stripe = Revenue
  AWS = Software and Subscriptions
  Gusto = Payroll
Or type 'none' to skip this for now - you can always add rules later."

Question 5:
"Where should I save your daily reports?
Provide a folder path, for example: ~/Desktop/bookkeeping-reports/
I will create this folder if it does not already exist."

After all answers are collected, save to ~/.bookkeeper-config.json and
output:

"Setup complete. Your AI Bookkeeper is configured for [platform].
Make sure you're logged into [platform] in Chrome before each run.
Type /bookkeeper anytime to run a manual session, or wait for your
scheduled daily run."

---

## DAILY RECONCILIATION

### Step 1 - Open your accounting platform in Chrome

Use the browser tools to navigate to the platform URL from config.
Wait for the dashboard to fully load before proceeding.

If a login screen appears: stop and display the ACTION REQUIRED message
from the Requirements Check section above.

### Step 2 - Navigate to uncategorized transactions

Follow the correct path for the user's platform:

QuickBooks Online:
- Click "Banking" or "Transactions" in the left sidebar
- Click the "For Review" tab
- Wait for the transaction list to load

Xero:
- Click "Accounting" in the top menu
- Click "Bank Accounts"
- Click "Reconcile [account name]"

Wave:
- Click "Accounting" in the left sidebar
- Click "Transactions"
- Filter by Status: Uncategorized

FreshBooks:
- Click "Accounting" in the left sidebar
- Click "Chart of Accounts" then "Transactions"
- Filter for unreviewed items

For any other platform: navigate to the section that shows new or
unreviewed bank transactions waiting to be categorized.

### Step 3 - Process each transaction

Work through the full list of uncategorized transactions one by one.

For each transaction:

  1. Check the auto-rules from config.
     - If the vendor name matches a rule (case-insensitive), apply that
       category directly inside the platform using the browser.
     - Mark the transaction as accepted/reviewed after categorizing.
     - Add to the COMPLETED list: date, vendor, amount, category applied.

  2. If no rule matches and the vendor name gives a clear indication of
     the category (e.g. "UBER EATS" is clearly Meals and Entertainment):
     - Apply the most logical category directly in the platform.
     - Add to the COMPLETED list with a note: "auto-categorized by AI."

  3. If the transaction is genuinely ambiguous - the vendor name gives
     no clear indication:
     - Do NOT guess. Do NOT apply a category.
     - Add it to the QUESTIONS list for the daily report.
     - Leave it uncategorized in the platform so the owner can handle it.

Keep a running count:
  - completed: list of categorized transactions
  - questions: list of transactions needing owner input
  - errors: any transactions where the browser action failed

### Step 4 - Generate the daily report

Create a report file at: [reports_folder from config]/[YYYY-MM-DD]-bookkeeping-report.html

The report must include all three sections below.

---
SECTION 1: WHAT WAS DONE TODAY
List every transaction that was successfully categorized.
Format each line as:
  [DATE]  |  [VENDOR]  |  [AMOUNT]  |  [CATEGORY APPLIED]

---
SECTION 2: QUESTIONS FOR YOU
List every transaction that could not be confidently categorized.
For each one:
  [DATE]  |  [VENDOR]  |  [AMOUNT]
  Question: What category should this be assigned to?

Below the list, include these instructions:
  "To answer these: Log into [platform] and go to [navigation path for
  uncategorized transactions]. These items are still in your For Review
  queue. Categorize them manually and mark as reviewed."

---
SECTION 3: SUMMARY
  Date: [today]
  Platform: [platform name]
  Transactions reviewed: [total count]
  Auto-categorized: [count]
  Questions for you: [count]
  Errors: [count]

---

### Step 5 - Open the report automatically

When the report file is saved, open it in Chrome so the owner sees it
immediately without having to find the file.

### Step 6 - Final output

Output to the Claude Code terminal:

"Bookkeeping complete for [date].
  Categorized: [n] transactions
  Questions for you: [n] (see report)
  Report opened in Chrome.

To trigger another session anytime, type /bookkeeper."

---

## RULES

- Never attempt to log in to any platform. If a login screen appears, stop.
- Never share or store passwords, credentials, or account numbers.
- Do not guess ambiguous categories - flag them for the owner instead.
- Apply auto-rules from config without asking - the owner already approved these.
- If a browser action fails (button not found, page error), log it in the
  errors section and continue with the remaining transactions.
- The goal is a clean, reliable daily report - not perfection on ambiguous items.

Frequently Asked Questions

What if I'm not always logged into QuickBooks?

The AI will stop and alert you. It will not try to log in on your behalf. In QuickBooks Online, check the "Keep me logged in" or "Remember this device" option when signing in - this keeps your session active for 30+ days and prevents this issue.

Can I use this with Xero, Wave, or FreshBooks instead of QuickBooks?

Yes. During first-time setup (Step 5), you choose your platform. The skill has navigation instructions built in for QuickBooks Online, Xero, Wave, and FreshBooks. If you use a different platform, it will ask you to describe where to find uncategorized transactions.

What if I want to add a new auto-categorization rule?

Open Claude Code, type /bookkeeper, and say "add a new auto-rule." Claude will ask you for the vendor name and category, update your config, and apply the rule from the next run forward.

Can I change what time it runs each day?

Yes. Open Claude Code and say: "Update my bookkeeper schedule to run at [new time] instead." Claude will update the schedule file and confirm the change.

What does it do with transactions it's not sure about?

It does not guess. Any transaction where the vendor name doesn't clearly indicate a category gets flagged in the "Questions For You" section of the daily report. It stays uncategorized in your platform until you handle it. This is intentional - wrong categories in your books are harder to fix than uncategorized ones.
