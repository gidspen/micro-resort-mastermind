The AI Bookkeeper: How to Reconcile Your Books Every Day with Claude Code

By Gideon Spencer · March 27, 2026

I posted a Reel about using the Claude Chrome extension as a daily bookkeeper - having it ask me questions, categorize transactions, and reconcile my books in real time. This is the step-by-step breakdown with everything you need to set it up yourself.

By the end of this guide you will have an AI skill installed in Claude Code that acts like a real bookkeeper: it asks what platform you use, works through uncategorized transactions one at a time, and saves a daily reconciliation log. No accounting background required.

What You're Building

A Claude Code skill that runs a daily bookkeeping session. It learns your accounting platform, applies your categorization rules automatically, and asks clarifying questions only when it genuinely doesn't know - exactly like a bookkeeper would. The goal: reconciled books every single day with real-time data.

What You Need

  • Claude Code - the terminal-based agent built into the Claude desktop app (claude.ai/download). Claude Pro ($20/month) or an API key required.
  • The Superpowers skills framework - a free, open-source library that gives Claude structured capabilities. You'll install this in Step 2.
  • Your accounting platform login - QuickBooks Online, Xero, Wave, FreshBooks, or anything else. The skill works with any platform.
  • Transaction exports - CSV downloads from your bank or accounting platform (the skill walks you through this).
  • About 20 minutes for initial setup. Daily sessions after that run 5-10 minutes.

Step 1

Download Claude and Open Claude Code

Go to claude.ai/download and install the Claude desktop app. Sign in with your Anthropic account.

You need a Claude Pro subscription ($20/month) to use Claude Code. Once logged in, open your terminal (Mac: press Cmd + Space, type "Terminal", hit enter) and type:

claude

Hit enter. Claude Code will launch. You're now talking directly to an AI agent that can read files, run commands, and install tools on your computer.

The Bookkeeper Skill File

This is the skill file Claude Code will install. Every instruction below gets saved to ~/.claude/skills/bookkeeper/SKILL.md and becomes available the next time you open Claude Code. Copy it using the button, or let Claude install it automatically using the command above.

---
name: bookkeeper
description: Daily bookkeeping reconciliation agent. Asks about your accounting platform on first run, categorizes transactions interactively, and produces a daily summary log.
trigger: /bookkeeper
---

# AI Bookkeeper Skill

You are acting as a meticulous daily bookkeeper. When this skill is invoked, follow this exact protocol.

---

## FIRST-TIME SETUP (run once, then skip)

Check whether ~/.bookkeeper-config.json exists.

If it does NOT exist, run the full setup sequence before doing anything else:

1. Ask: "What accounting platform do you use? (QuickBooks Online / Xero / Wave / FreshBooks / other - please specify)"
2. Ask: "How will you provide your transactions each session? Options: (a) I'll paste them directly into this chat, (b) I'll provide a CSV file path, (c) I'll drag and drop the CSV"
3. Ask: "What is your primary business currency?"
4. Ask: "Are there any vendors or transaction sources you want auto-categorized without being asked? For example: 'Stripe = Revenue', 'AWS = Software & Subscriptions', 'Gusto = Payroll'. List as many as you like, or say 'none'."
5. Ask: "What categories does your accounting platform use? List your top 5-10, or say 'use standard categories' and I'll apply QuickBooks-style defaults."

Save all answers to ~/.bookkeeper-config.json in this format:
{
  "platform": "...",
  "input_method": "...",
  "currency": "...",
  "auto_rules": { "vendor_name": "Category", ... },
  "categories": ["Revenue", "Cost of Goods", "Payroll", ...]
}

Confirm: "Setup complete. Your bookkeeper is configured. Type /bookkeeper anytime to start your daily session."

---

## DAILY RECONCILIATION SESSION

Load ~/.bookkeeper-config.json.

1. Greet the user briefly: "Ready for today's reconciliation. [Date]. How would you like to provide today's transactions?"

2. Accept transactions via the method configured in setup (paste, CSV path, or drag-and-drop).

3. Apply auto-categorization rules from config first - do not ask about these.

4. For each remaining uncategorized transaction, ask ONE clear question:
   - Format: "Transaction: [VENDOR] - $[AMOUNT] on [DATE]. Is this [most likely Category A] or [most likely Category B]? (Or type the correct category)"
   - Give 2 specific options based on the vendor name - make an educated guess
   - Do not ask vague questions like "what category is this?" - always suggest options

5. If a transaction is ambiguous and you cannot make a reasonable guess, flag it:
   - Format: "FLAGGED FOR REVIEW: [VENDOR] $[AMOUNT] - I couldn't determine the category. You can categorize this manually in [platform]."

6. After all transactions are processed, produce a reconciliation summary:

---
DAILY RECONCILIATION SUMMARY - [DATE]
--------------------------------------
INCOME
  [Category]: $[amount]
  Total Income: $[total]

EXPENSES
  [Category]: $[amount]
  [Category]: $[amount]
  Total Expenses: $[total]

NET: $[income - expenses]

TRANSACTIONS PROCESSED: [count]
AUTO-CATEGORIZED: [count]
MANUALLY CATEGORIZED: [count]
FLAGGED FOR REVIEW: [count] (see details below)

FLAGGED ITEMS:
  - [VENDOR] $[AMOUNT] [DATE]
---

7. Save this summary to ~/bookkeeping-logs/[YYYY-MM-DD].md (create the directory if it doesn't exist).

8. Ask: "Reconciliation complete. Would you like me to summarize anything by category, or are you done for today?"

---

## RULES

- NEVER ask for banking passwords, login credentials, or full account numbers
- Work from pasted data or CSV exports only - never request direct bank access
- When guessing a category, always give the user 2 options - do not make them type from scratch
- Flag rather than guess when genuinely uncertain
- Keep questions short - one transaction, one question, move on
- If the user says "that's wrong" about a category, update your guess and ask if they want to add it as a permanent auto-rule
- At the end of each session, remind the user to sync the categorized transactions back to their accounting platform manually (or via their platform's import feature)

Step 4: Set It Up to Run Every Day

The bookkeeper only works if you actually open it daily. Here are three options depending on how technical you want to get:

Option A: Set a Phone Reminder (Easiest)

Set a daily alarm for a consistent time - end of business day works well. When it goes off, open Claude Code and type /bookkeeper. That's it. The habit is more important than the automation.

Option B: Desktop Shortcut

Create a shell script that opens Claude Code and types the trigger for you. Paste this into Claude Code to create it:

Create a shell script at ~/run-bookkeeper.sh that opens a new terminal window and runs Claude Code with the /bookkeeper command. Make it executable. Then create a macOS Automator app from it so I can double-click it from my desktop.

Option C: Scheduled Daily Task (Most Automated)

Use macOS launchd to run a daily reminder at a set time. Paste this into Claude Code:

Create a macOS launchd plist that runs at 5:00 PM every weekday and sends me a macOS notification that says "Time to reconcile your books - open Claude Code and type /bookkeeper". Install it to ~/Library/LaunchAgents/ and load it.

Recommended Approach

Start with Option A. Build the habit for two weeks, then automate. Most people who try to automate first never actually do the thing - the reminder exists but they ignore it. The habit comes first.

Syncing Back to Your Accounting Platform

The bookkeeper skill categorizes your transactions inside Claude and saves a daily log. It does not directly write back to QuickBooks, Xero, or your bank. Here's how to complete the loop:

  • QuickBooks Online: The categorized summary becomes your reference. Match it against QBO's "For Review" tab and apply categories there manually. Takes 3-5 minutes once you know what each transaction is.
  • Xero: Same approach - use the reconciliation summary as your source of truth, then confirm matches in Xero's bank reconciliation screen.
  • Wave: Use the Claude summary to batch-categorize transactions in Wave's transaction list.
  • Any platform with CSV import: Ask Claude to export the categorized transactions as a CSV after each session, then import that file into your platform.

To get a CSV export from your session, type this at the end of your daily reconciliation:

Export today's categorized transactions as a CSV file to ~/bookkeeping-logs/[today's date]-export.csv in a format compatible with QuickBooks Online import.

What the Daily Log Looks Like

Every session saves a Markdown file to ~/bookkeeping-logs/. After 30 days, you have a complete running record of your business finances - organized, categorized, and searchable. You can ask Claude to summarize a week, a month, or flag anomalies:

Read all files in ~/bookkeeping-logs/ from the past 30 days and give me a monthly P&L summary broken down by category.
Look through my ~/bookkeeping-logs/ and flag any vendors that appear for the first time this month - I want to review anything unusual.
Security - Read This First

What Not to Share With Claude

Claude Code processes conversations through Anthropic's servers. It is not the right place for:

  • Banking login credentials or passwords
  • Full account or routing numbers
  • Social Security numbers or tax IDs
  • Client financial data or personally identifiable information (PII)
  • API keys with financial write access (Stripe secret keys, etc.)

The bookkeeper skill is designed to work from CSV exports and pasted transaction data only - not direct bank access. Think of it as handing your bookkeeper a printout of your bank statement, not your login.
